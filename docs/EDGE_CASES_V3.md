# Edge Cases V3

## Quotation / ขอใบเสนอราคา
- ส่งฟอร์มโดยไม่มีรูป: อนุญาตหรือแจ้งเตือนตาม policy
- อัปโหลดเกิน 10 รูป ต้อง reject
- รูปเกิน 5MB ต้อง reject
- map_link ว่าง แต่ address มีข้อมูล ต้องบันทึกได้
- latitude/longitude ว่าง ต้องบันทึกได้
- latitude/longitude นอกช่วง ต้อง reject
- เบอร์โทรซ้ำ ต้องใช้ customer เดิม ไม่สร้างซ้ำโดยไม่จำเป็น
- tracking code + phone ไม่ตรง ต้องไม่แสดงข้อมูลลูกค้าคนอื่น

## Portfolio / ผลงาน
- ไม่มีรูปปก ให้ fallback เป็น placeholder
- มีรูปก่อนทำแต่ไม่มีรูปหลังทำ ต้องแสดงส่วนที่มีและ placeholder ส่วนที่ว่าง
- ลบผลงานที่มีรูป ต้อง cascade หรือจัดการ storage ตาม policy
- ตั้งรูปปกใหม่ ต้องยกเลิก is_cover ของรูปเดิม
- รูปประเภทอื่น ๆ ไม่ควรปนใน Before/Progress/After
- เปลี่ยนภาษา EN แต่ข้อมูล EN ว่าง ให้ fallback ภาษาไทย

## Work Order / ใบงาน
- สร้างใบงานจากใบเสนอราคาซ้ำ ต้องเปิดใบงานเดิม
- พิกัดว่าง แต่มี Google Maps Link ต้องบันทึกได้
- ปิดใบงานแล้วสร้างผลงาน ต้องดึงรูปตามประเภท before/during/after
- ไม่มี service_id ต้องสร้างใบงานได้ แต่แสดงเป็นไม่ระบุ

## Backup / Export
- ไม่มีข้อมูล export ต้องดาวน์โหลดไฟล์เปล่าที่ไม่ error
- ข้อมูลไทยต้องไม่เพี้ยนเมื่อเปิดแบบ UTF-8
- Backup JSON ต้องไม่รวมรหัสผ่าน user
- Backup รูปภาพจริงควรตรวจ storage permission ก่อนทำ ZIP

## UX / Accessibility
- ปุ่มต้องสูงพอและตัวหนังสือไม่ชิดขอบบน
- ปุ่มเลื่อนขึ้น/ลงต้องมีข้อความ ไม่ใช่ไอคอนอย่างเดียว
- มือถือควรเห็นปุ่ม โทร / LINE / ขอราคา ตลอด
